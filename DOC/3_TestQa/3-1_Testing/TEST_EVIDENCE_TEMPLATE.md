# 🧾 SC04_01_A 日報バッチ承認性能試験結果報告書

## ■ 試験目的
日報承認API（`/production/daily-report/approve`）の性能を検証する。  
複数ユーザー（5名）が同時に承認操作を実行した場合の応答時間およびシステム処理影響を確認する。

---

## ■ 試験条件
| 項目 | 内容 |
|------|------|
| 試験環境 | dev-soyokanri.crj.dev.jems-cloud.jp |
| 試験方式 | k6 によるシナリオテスト |
| 同時仮想ユーザ数 (VUs) | 5 ユーザ |
| 試験データ | `registration_step` テーブルの未承認（status=1）データ5件 |
| 実行コマンド |  
`docker run --rm -i -e TARGET_URL="http://35.78.163.192:3000" -e USERNAME="jems-a@email.com" -e PASSWORD="pass1234" -v "%cd%":/k6 grafana/k6 run /k6/k6-tests/sc04/sc04_01_a_nippobatch.js` |

---

## ■ 試験シナリオ概要
1. ログインAPI (`/auth/login`) にてセッションCookieを取得  
2. 各VUが日報承認API (`/production/daily-report/approve`) を同時実行  
3. 登録ステップ (`registration_step`) のステータスを「1 → 5（承認済）」に更新  
4. バッチジョブ (`dailyReportGenerationJob`) が自動起動されることを確認  

---

### 実行コマンド

```
docker run --rm -i -e TARGET_URL="http://35.78.163.192:3000" -e USERNAME="jems-a@email.com" -e PASSWORD="pass1234" -v "%cd%":/k6 grafana/k6 run 
/k6/k6-tests/sc04/sc04_01_a_nippobatch.js


```

| **P99 応答時間** | N/A | ✅ 合格 |
| **最大応答時間** | 428.93 ms | ✅ 合格 |
| **エラー率** | 0% | ✅ 合格 |
| **総リクエスト数** | 5件 | ー |

---

## ■ ログ確認結果（Appサーバ）
- `PUT /production/daily-report/approve` が 5回正常完了  
- 各リクエストで `registration_status` = 1 → 5 に更新  
- バッチ起動 (`dailyReportGenerationJob`) の呼び出しを確認  
- Exception / SQLエラー発生なし  

---

## ■ 考察
- 全ユーザーが同時承認しても処理遅延やDBロック競合は発生せず。  
- 平均応答時間は 0.4 秒台と非常に安定しており、SLO（<30秒）を大幅に満たしている。  
- 同時実行時のエラーやスレッド詰まりも検出されなかった。
