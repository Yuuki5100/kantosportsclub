
# 📄 Subtree統合運用ドキュメント（完全版）

> 文書ステータス: 補足
> 位置づけ: `crj-project` 連携時の Subtree 運用資料
> 参照元: `開発ルール総則.md`

（CRJ-project × common-architecture）

## 🎯 目的

本ドキュメントは、`common-architecture` リポジトリの `develop-crj` ブランチと `crj-project` の並行開発・統合を安全かつ効率的に行うための **Git Subtree 運用ルール**を定義します。
特に `crj-project` 側で**共通コードをルート直下に展開し、マージ管理可能な状態で取り込む**ことを目的としています。

---

## 📦 対象構成

| リポジトリ                 | 用途         | ブランチ                                 |
| --------------------- | ---------- | ------------------------------------ |
| `common-architecture` | 共通コード基盤    | `develop-crj`                        |
| `crj-project`         | 個別プロジェクト本体 | `main`, `feature/*`, `subtree-merge` |

---

## 🪜 運用フロー概要

```text
common-architecture (develop-crj)
         ↓ subtree pull
CRJ-project (subtree-merge)
         ↓ 展開
CRJ-project (feature/xxx や main にマージ)
```

---

## 🛠 初期設定手順（1回のみ）

### 1.1 リモート登録（済みの場合はスキップ）

```bash
git remote add common-architecture https://ide-gitlab.j-ems.jp/common-archetecture/common-archetecture.git
```

---

## 🚀 開発統合フロー

### STEP 1: subtree-merge ブランチを作成

```bash
git checkout -b subtree-merge
```

---

### STEP 2: 共通コード（develop-crj）を subtree 経由で取り込み

```bash
git fetch common-architecture
git subtree add --prefix=common-architecture common-architecture develop-crj --squash
```

> ※2回目以降は `add` ではなく `pull` を使用（後述）

---

### STEP 3: 共通コードをルート直下に展開（PowerShell / Git Bash 両対応）

#### ✅ PowerShell の場合：

```powershell
Copy-Item -Recurse -Force -Path .\common-architecture\* -Destination .
Remove-Item -Recurse -Force .\common-architecture\
git add .
git commit -m "展開: common-architecture/develop-crj をルートに反映"
```

#### ✅ Git Bash（Linux/macOS）の場合：

```bash
cp -ru common-architecture/* .
rm -rf common-architecture/
git add .
git commit -m "展開: common-architecture/develop-crj をルートに反映"
```

---

### STEP 4: プロジェクト開発ブランチへマージ

```bash
git checkout feature/your-feature
git merge subtree-merge
```

> 🔧 コンフリクトが発生した場合は手動で解決 → `git commit`

---

## 🔄 共通コードの更新取り込みフロー（2回目以降）

```bash
git checkout subtree-merge
git fetch common-architecture
git subtree pull --prefix=common-architecture common-architecture develop-crj --squash
```

### 取り込み後、再度ルート展開：

#### ✅ PowerShell：

```powershell
Copy-Item -Recurse -Force -Path .\common-architecture\* -Destination .
Remove-Item -Recurse -Force .\common-architecture\
git add .
git commit -m "更新: develop-crj の最新をルートに展開"
```

#### ✅ Git Bash：

```bash
cp -ru common-architecture/* .
rm -rf common-architecture/
git add .
git commit -m "更新: develop-crj の最新をルートに展開"
```

---

## 📌 注意事項

| 項目                        | 内容                                            |
| ------------------------- | --------------------------------------------- |
| `--squash` を使う理由          | 履歴を1コミットに圧縮し、差分を管理しやすくするため                    |
| subtree-merge は rebase 禁止 | squash履歴と不整合になるため                             |
| コンフリクト時の対応                | `diff`, `git status`, `git mergetool` などで手動解決 |
| 展開先のディレクトリ構造              | 常にルート構成を一致させておくこと（開発整合性のため）                   |
