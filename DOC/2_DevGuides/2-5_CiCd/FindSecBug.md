# SpotBugs / FindSecurityBugs セキュリティチェック運用ルール
## 1. SpotBugs レポートの確認方法

- セキュリティやコード品質のチェック結果は、Maven実行後に以下のxmlファイルで確認できます：

appserver/target/spotbugs.xml

## 2. SpotBugs 実行タイミングルール

通常のアプリ起動やデバッグ時に SpotBugs を毎回実行する必要はありませんが、以下のタイミングでは必ず実行してください。

###  実行が必要なタイミング

- 自分のタスクで **ソースコードを新規追加/修正** した場合
- **MR提出前**

## 3. 各開発者の修正責任範囲
-各開発者は、自分のタスクや改修箇所に関連して発生した SpotBugs の指摘に対して責任を持って対応する。

-他の開発者が書いた／変更したコードによって出たバグについては、基本的に対応不要。

- マージやレビューの前に SpotBugs のレポートを確認し、該当バグが自分の修正によるものでないかを確認すること。

## 4. バグの重大度と対応ルール

| レベル      | 説明                                  | 対応ルール                     |
|-------------|---------------------------------------|--------------------------------|
| **High**    | 明らかなセキュリティ脆弱性、バグ     | ✅ **必ず修正する**             |
| **Medium**  | 保守性や安全性にやや問題がある        | ⏱ **可能なら修正（時間がある場合）** |
| **Low**     | 軽微な注意点（ログ形式など）           | 📄 **優先度は低（放置OK）**     |


## 5. SpotBugsの実行手順

以下の手順で SpotBugs を実行し、HTMLレポート（spotbugs.html）を生成してください。

---

### 🔁 実行対象：appserver モジュールの場合

#### ✅ ステップ 1：ルートディレクトリでコンパイル

- No1. BE プロジェクト全体の `.class` ファイルを生成します。

```bash
cd BE
mvn clean compile
```
- No2. appserver モジュールに移動してspotbug xmlを作成
cd appserver
```
mvn spotbugs:spotbugs -Dspotbugs.outputFormat=xml
```
