# 環境構築・実行手順書（現行運用）

本書は、当プロジェクトの現行開発環境（Dev Container + Docker）を前提にした手順をまとめたものです。

## 1. 前提環境

| 種別 | 推奨 | 備考 |
|---|---|---|
| OS | WSL2 + Ubuntu 22.04 / 24.04 | リポジトリは WSL 側に配置 |
| Docker | Docker Engine + Compose V2 | `docker compose` が利用可能であること |
| VS Code | Dev Containers 拡張 | 任意（CLI運用のみでも可） |
| Git | 最新安定版 | |

補足:

- Node.js / Java / Maven / Python は Dev Container と Docker イメージで統一運用する。
- `nvm` は不採用（Dockerfile 固定版との二重管理を避けるため）。
- サービス起動やデータ取得は compose 定義を正とし、実行入口は `docker/stack.sh` に統一する。

## 2. 推奨起動手順（modern family）

```bash
cd /home/{user}/work/common-archetecture
bash docker/stack.sh --family modern multi-container --logs
```

`--family` を省略した場合は `modern` が選択されます。

## 3. テスト/検証コマンド

```bash
# Frontend lint + unit test
docker/stack.sh test-fe

# Backend test
docker/stack.sh test-be

# E2E
docker/stack.sh e2e

# API contract smoke (Newman)
docker/stack.sh newman

# External API mock (WireMock)
docker/stack.sh wiremock

# OpenTelemetry Collector + InfluxDB + Grafana (monitoring)

# 集計対象一覧: DOC/2_DevGuides/2-1_HowToDevelop/05_otel_aggregation_targets.md

docker/stack.sh python

docker/stack.sh k6
docker/stack.sh zap-baseline
```

## 4. 系統切替（csharp / java-legacy）

family 切替前に、既存 family を停止:

```bash
docker/stack.sh --family modern down
```

```bash
docker/stack.sh --family csharp up
docker/stack.sh --family csharp status
docker/stack.sh --family csharp down
```

```bash
docker/stack.sh --family java-legacy up
docker/stack.sh --family java-legacy status
docker/stack.sh --family java-legacy down
```

運用ルール:

- family は当面「排他的運用」（同時起動しない）。
- `csharp` は .NET 8 SDK テンプレートを提供。
- `java-legacy` は Apache + Tomcat 分離テンプレートを提供（`dist/java-legacy/tomcat/webapps/` に配備）。
- `java-legacy` の標準構成は `Java 1.8.0_202` + `Tomcat 6.0.43`。
- `java-legacy` 起動前に `docker/java-legacy/assets/` へ JDK tarball（推奨: `jdk-8u202-linux-x64.tar.gz`）を配置する。
- 必要に応じて `6.zip`（Tomcat 一式）を同ディレクトリへ配置する。

## 5. タイムゾーン方針

- Dev Container / Docker サービスは `Asia/Tokyo` に統一。
- MySQL は `--default-time-zone=+09:00` を設定。
- Java サービスは `-Duser.timezone=Asia/Tokyo` を設定。
- アプリの DB 接続設定（例: `connectionTimeZone=UTC`）は現時点では変更対象外。

## 6. フロントエンド API 接続方針（Docker）

- フロントエンドは Gateway 経由で API に接続する。
- Docker 内既定値: `NEXT_PUBLIC_API_BASE_URL=http://gateway:8888`

## 7. 参考（非推奨の直起動）

ローカルに直接 Node.js / Java / MySQL を入れての起動は可能だが、現行の標準運用ではない。環境差分による不具合回避のため、原則 Docker/Dev Container を利用する。
