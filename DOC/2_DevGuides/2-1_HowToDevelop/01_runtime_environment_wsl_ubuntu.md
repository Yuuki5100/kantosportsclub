# 実行環境（WSL + Ubuntu）

## 想定構成

- ホスト OS: Windows 11
- Linux 実行基盤: WSL2
- ディストリビューション: Ubuntu 22.04 / 24.04
- 開発場所: WSL 側の Linux ファイルシステム（例: `/home/<user>/work/...`）

## この構成を推奨する理由

- Docker と開発ツールを Linux 側に寄せることで、動作差異を減らしやすい
- パス・権限・改行コードのトラブルを減らしやすい
- Docker / Node / Java 系の I/O 性能が安定しやすい

## 注意点

- リポジトリは `C:` 配下（`/mnt/c/...`）ではなく、WSL 側に配置する
- Docker を使う場合は WSL 連携を有効化した環境を使う
- VS Code を使う場合は WSL リモート接続で作業する
