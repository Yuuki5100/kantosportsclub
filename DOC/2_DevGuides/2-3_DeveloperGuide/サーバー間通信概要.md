
# 📡 サーバー間通信構成概要ドキュメント（操業管理 ↔ 入出庫管理）

## 🧭 概要

本ドキュメントは、操業管理アプリケーションから入出庫管理アプリケーションへ行われる**双方向DB同期処理に関するサーバー間通信構成**の全体像を示す。  
対象は、**appserver（操業）→ appserver（入出庫）** 間での通信に関連する **gateway経路・通信制御・認証方式・ルート設計** である。

---

## 🗺️ 通信経路図（リクエストルート）

```text
[ブラウザ（操業ユーザー）]
   ↓ HTTPS
[nginx（操業）]                         ← 外部公開 /api/update-shared-table
   ↓
[gateway（操業）]                       ← 内部ルーティング
   ↓
[appserver（操業）]                     ← 操業DB更新 + 入出庫API呼び出し
   ↓ HTTPS（内部HTTPクライアント）
[gateway（操業）]                       ← 再び gateway を経由して外向きに出る
   ↓
[gateway（入出庫）]                    ← VPC間 or FQDN接続
   ↓
[appserver（入出庫）]                  ← 入出庫DBを更新する受信エンドポイント
````

---

## 🔒 通信制御とセキュリティ設定

### ✅ 通信経路制限

| 区間                        | 制御方式                                         |
| ------------------------- | -------------------------------------------- |
| gateway操業 → gateway入出庫    | IP制限（操業VPC CIDR のみ許可） or VPC Peering         |
| appserver操業 → gateway操業   | http\_proxy 経由で構成（明示的ルートを通す）                 |
| gateway入出庫 → appserver入出庫 | 内部リバースプロキシで `/api/sync-shared-table` にルーティング |

### ✅ 認証・認可

* **通信方式**：HTTPS（TLS 1.2以上）
* **認可方式（入出庫側）**：

  * HMAC署名（例：`X-Signature`, `X-Timestamp`, `X-Nonce`）
  * または短期APIキーによるトークン認証（Bearer token）
* **リクエストヘッダ例**：

```http
POST /api/sync-shared-table
X-Request-Id: abc123
X-Timestamp: 20250805T010000Z
X-Signature: hmac_sha256(...)
Content-Type: application/json
```

## 🌐 DNS / 通信ルート構成

| 項目            | 内容                                                             |
| ------------- | -------------------------------------------------------------- |
| 入出庫API呼び出しURL | `https://inventory-api.internal.company/api/sync-shared-table` |
| 名前解決          | gateway操業 から gateway入出庫 の FQDN を内部DNSで解決                       |
| 経路制御          | 社内 http\_proxy / VPC Peering / PrivateLink のいずれかを使用            |

---

## 📁 操業管理アプリ内の通信構成（概要）

| モジュール                   | 内容                                      |
| ----------------------- | --------------------------------------- |
| `SyncRequestSender`     | `RestTemplate` / `WebClient` 経由でHTTPS通信 |
| `SyncRequestLogger`     | 通信前後のログ出力（requestId単位）                  |
| `HmacSignatureUtil`     | シグネチャ生成および検証用の共通ライブラリ                   |
| `PendingSyncRepository` | 通信失敗時の再送エントリ保存用テーブル                     |

---

## ✅ ログ・トレーサビリティ設計

| ログ対象         | 内容例                                       |
| ------------ | ----------------------------------------- |
| 操業→入出庫呼び出しログ | requestId, 送信先URL, HTTP status, 送信bodyの要約 |
| 受信ログ（入出庫側）   | requestId, 検証済み署名, 対象DB更新結果               |
| エラー通知（任意）    | Slack/Teams 通知 or アラートログ記録（再送失敗など）        |

---

## 🚧 想定される通信エラーと対策

| エラー          | 対策                                 |
| ------------ | ---------------------------------- |
| タイムアウト       | リトライ設計（指数バックオフ） + 失敗ログ記録           |
| 認証エラー（署名不一致） | 操業appserverと入出庫appserverで共通鍵管理を明示化 |
| DNS解決失敗      | 内部向け名前解決（internal DNS or hosts）確認  |
| 通信先拒否（403）   | gateway入出庫のIPホワイトリスト / トークン設定確認    |
