# ✅ ファイル連携機能テスト仕様書（バックエンド）

## 1. 概要

本仕様書は、`FileImportController` を中心とした**ファイルアップロード・バリデーション・DB登録処理**に対する単体テスト・統合テスト仕様を定義します。正常処理・バリデーションエラー・例外発生時の振る舞いなどを網羅し、安定したファイルインポート機能の担保を目的とします。

---

```
src/test/java

    └── service

        └── FileNameResolverTest.java
```

## 3. 対象クラス一覧

| クラス                 | 優先度 | 主なテスト観点                                           |
|----------------------|--------|----------------------------------------------------------|
| `FileImportController` | ⭐⭐⭐    | ファイル受信・レスポンス制御・バリデーション結果の出力       |
| `ImportJobExecutor`   | ⭐⭐☆    | 処理分岐（ファイル種別判定、保存、検証、書き込みの統括）     |
| `FileSaver`           | ⭐⭐☆    | S3/ローカル保存の分岐、保存成否、エラーハンドリング          |
| `FileNameResolver`    | ⭐⭐☆    | ファイル名からのメタ情報抽出（日付・種別）                  |
| `FileTypeResolver`    | ⭐⭐☆    | 拡張子・中身からのファイル種別識別                         |

## 4. テスト仕様

### 4.1 FileImportControllerTest

| No | エンドポイント      | テスト内容                                            | 期待される結果                             |
|----|---------------------|-----------------------------------------------------|------------------------------------------|
| 1  | `POST /import`      | ファイル・filename 両方あり、正常なファイル         | 200 OK + 成功メッセージ                  |
| 2  | `POST /import`      | filename が空文字・null                             | 400 BadRequest + エラーコード             |
| 3  | `POST /import`      | Multipart ファイルが null                            | 400 BadRequest + エラーコード             |
| 4  | `POST /import`      | 保存処理で例外発生（IOException など）               | 500 Internal Server Error + エラーメッセージ |
| 5  | `POST /import`      | バリデーションエラーあり                             | 200 OK（エラー行あり） + ログに警告      |
| 6  | `POST /import`      | 不正なファイル種別（サポート対象外）                | 400 BadRequest + エラーコード             |

---

### 4.2 ImportJobExecutorTest

| No | メソッド            | テスト内容                                      | 期待される結果                             |
|----|---------------------|-----------------------------------------------|------------------------------------------|
| 1  | `runJob()`          | 正常なファイル種別・内容                       | バリデーション→保存→書き込みまで実行     |
| 2  | `runJob()`          | 不正なファイル内容                             | エラー件数に応じたログ出力・DB書き込みスキップ |
| 3  | `runJob()`          | ファイルタイプが未解決                         | 例外またはスキップ                        |
| 4  | `runJob()`          | 書き込み中に RuntimeException 発生             | キャッチしてログ出力、エラー結果に反映   |

---

### 4.3 FileSaverTest

| No | メソッド            | テスト内容                                      | 期待される結果                             |
|----|---------------------|-----------------------------------------------|------------------------------------------|
| 1  | `save()`            | S3保存が有効（S3StorageService）               | 正しくアップロードされ、パスが返却される   |
| 2  | `save()`            | ローカル保存が有効（LocalStorageService）      | ファイルが保存され、ファイル名が返却される |
| 3  | `save()`            | 例外が発生するケース（IOException）            | RuntimeExceptionとして再スロー            |

---

### 4.4 FileNameResolverTest

| No | メソッド            | テスト内容                                      | 期待される結果                             |
|----|---------------------|-----------------------------------------------|------------------------------------------|
| 1  | `resolve()`         | `users_20240101.csv` のような正常ファイル名     | 種別=users、日付=2024-01-01 として抽出     |
| 2  | `resolve()`         | フォーマット不正なファイル名                   | IllegalArgumentException スロー           |

---

### 4.5 FileTypeResolverTest

| No | メソッド            | テスト内容                                      | 期待される結果                             |
|----|---------------------|-----------------------------------------------|------------------------------------------|
| 1  | `resolve()`         | `.csv` 拡張子の場合                           | ファイルタイプが CSV                      |
| 2  | `resolve()`         | `.xlsx` 拡張子の場合                          | ファイルタイプが EXCEL                   |
| 3  | `resolve()`         | 未知の拡張子（例: `.txt`）                    | IllegalArgumentException スロー           |

## 6. サンプルレスポンス例

### ✅ 正常アップロード完了
```json
{
  "success": true,
  "data": "✅ users_20240401.csv インポート成功",
  "error": null
}
```

### ❌ ファイル種別エラー
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "E400",
    "message": "不正なファイル種別です"
  }
}
```

### ❌ 保存失敗（IOException）
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "E500",
    "message": "ファイルの保存に失敗しました"
  }
}
```

## 7. カバレッジ補足

- 保存処理（ローカル/S3）の両方を想定
- バリデーション結果をログ出力・レスポンスで返却する動作の確認
- ファイル種別毎の切り替えロジック、および拡張性の担保
- nullチェック、空文字チェック、例外発生系など入力妥当性も網羅
