# 📂 フォルダ構成 設計書

本ドキュメントでは、Next.js アプリケーションのフォルダ構成と各ディレクトリの役割について説明します。  
プロジェクトの統一的な構成を維持し、開発の効率化と保守性向上を目的とします。

---

## 1. 🎯 目的
### **1-1. 概要**
本フォルダ構成は、Next.js アプリケーションにおける **「モジュールの責務を明確にし、適切な分離を行う」** ことを目的としています。

- **保守性の向上**：役割ごとにフォルダを整理することで、可読性と開発効率を向上させる。
- **スケーラビリティの確保**：機能追加や拡張が容易になる。
- **コードの再利用性向上**：共通のコンポーネントやユーティリティを明確に管理し、重複を防ぐ。

## 2. 📌 設計方針

### 2-1. アーキテクチャ方針

- **責務の分離**  
  各フォルダが特定の責務を持ち、それぞれの役割を明確にする。

- **統一された命名規則**
  - ディレクトリ名は **ローワーキャメルケース（例：`test-utils`）**を使用。
  - ファイル名は **PascalCase（例：`ErrorBoundary.tsx`）** を使用。

- **ユニットテストの分離**  
  - 各機能のテストは `__tests__` ディレクトリ内に配置し、実装ファイルと分離する。
- **注釈**
  本設計書では各ディレクトリの責務範囲を明確にするため、ルートディレクトリに配置された各種のconfig.tsについての記述は割愛します。

## 3. 📂 フォルダ構成とファイルの役割

```plaintext
├─ public/                # 静的ファイル（favicon, robots.txt など）
├─ scripts/               # 開発・デプロイ時のスクリプト
└─ src/
    ├─ api/               # API 通信関連のモジュール
    │  └─ services/       # API サービス定義（REST, GraphQL など）
    │   ├─ v1/            # v1エンドポイントのサービス定義
    │   └─ v2/            # v2エンドポイントのサービス定義
    ├─ assets/            # 静的アセット（フォント、画像、スタイル）
    │  ├─ fonts/          # Web フォントファイル
    │  ├─ images/         # 画像ファイル
    │  └─ styles/         # グローバルスタイル（CSS, SCSS, Tailwind など）
    ├─ components/        # 再利用可能な UI コンポーネント
    │  ├─ base/           # 基本的な UI 部品（ボタン、入力欄など）
    │  ├─ composite/      # base コンポーネントを組み合わせたコンポーネント
    │  └─ functional/     # 特定のロジックを持つコンポーネント
    │      └─ __tests__/  # functional コンポーネントのテストコード
    ├─ config/            # 環境変数やアプリの設定ファイル
    ├─ hooks/             # React カスタムフック（useApi,useSnackbarなど）
    │  └─ __tests__/      # hooks のテストコード
    ├─ pages/             # Next.js のルーティング用ページ(各ページはディレクトリ毎にまとめ、langなどと一緒に管理)
    │  ├─ admin/          # 管理者向けページ
    │  ├─ login/          # 認証関連ページ
    │  ├─ myPage/         # ユーザー個別ページ
    │  ├─ user/           # 一般ユーザーページ
    │  └─ __tests__/      # ページ関連のテストコード
    ├─ slices/            # Redux Toolkit のスライス（状態管理）
    │  └─ __tests__/      # slices のテストコード
    ├─ test-utils/        # テスト用のユーティリティ関数
    ├─ types/             # TypeScript の型定義（DTO, インターフェース）
    └─ utils/             # 共通のユーティリティ関数
```

## 4. 📌 各ディレクトリの詳細

### 🔹 `public/`
- アプリケーションの **公開ファイル** を格納。
- 例: `favicon.ico`, `robots.txt`, `manifest.json` など。

### 🔹 `scripts/`
- 開発・デプロイ時に使用する **シェルスクリプト, Node.js スクリプト** を格納。
- 例: **DBマイグレーションスクリプト、ビルド・デプロイスクリプト、テンプレート作成スクリプト** など。

### 🔹 `src/api/`
- API 通信関連のモジュールを管理。

### 🔹 `src/api/services`
- API 通信関連のサービス定義。
- `/v1,/v2・・・` 内でバージョン毎にサービスを管理

### 🔹 `src/assets/`
- フロントエンドで使用する **静的アセット** を管理。
  - `fonts/`: **Webフォント**（woff2, ttf など）
  - `images/`: **画像リソース**（アイコン、バナーなど）
  - `styles/`: **SCSS, Tailwind CSS, Global CSS** など

### 🔹 `src/components/`
- **UI コンポーネントを責務ごとに分類**
  - `base/`: 最小単位の **UI コンポーネント**（ボタン、テキストフィールドなど）
  - `composite/`: 複数の **base** を組み合わせたコンポーネント
  - `functional/`: **各ページよりも上のレイヤーで呼ばれ、表示制御などの機能を持つコンポーネント**

### 🔹 `src/config/`
- 環境変数・アプリケーション設定の管理。

### 🔹 `src/hooks/`
- React カスタムフック（`useState`, `useEffect` など)を定義
- 例: `useAuth.ts`, `useFetch.ts`, `useForm.ts`

### 🔹 `src/pages/`
- Next.js のルーティング用のページ。
- **ディレクトリ構造が URL パスと一致** する。
```plaintext
user/
    ├── index.ts        # エントリーポイント
    ├── user.lang.ts    # 多言語対応用の設定ファイル
    ├── user.tsx        # ユーザー関連の React コンポーネント
```

### 🔹 `src/slices/`
- **Redux Toolkit のスライス（状態管理用のロジック）。**
- 例: `authSlice.ts`, `snackbarSlice.ts`, `errorSlice.ts`

### 🔹 `src/test-utils/`
- **Jest, Testing Library などで使用するテスト用ユーティリティ関数を管理。**

### 🔹 `src/types/`
- **TypeScript の型定義（DTO, APIレスポンス型など）。**

### 🔹 `src/utils/`
- **共通のユーティリティ関数（文字列操作、日時フォーマットなど）。**
