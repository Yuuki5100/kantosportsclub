# ErrorNotification モジュール仕様書

## 1. モジュール概要

### 1-1. 目的
`ErrorNotification` は、アプリケーション共通のエラー通知を画面上部に表示するコンポーネントである。`useError()` で保持しているエラーメッセージを監視し、自動消去付きの簡易通知として表示する。

### 1-2. 適用範囲
- 画面共通のエラー表示
- API 呼び出し失敗時の通知
- フォーム送信失敗や業務エラーの簡易表示

---

## 2. 設計方針

### 2-1. アーキテクチャ
- `useError()` から `errorMessage` と `clearError` を取得する。
- エラーメッセージがある間のみ通知 UI を表示する。
- 一定時間後に `clearError()` を呼び、自動で非表示にする。

### 2-2. 統一ルール
- 表示位置は画面上部中央固定とする。
- 背景色は赤、文字色は白とする。
- エラーメッセージが未設定のときは何も描画しない。
- 手動クローズボタン `×` を常に表示する。

## 4. コンポーネント仕様

**使用状態:**
- `errorMessage` - 表示対象エラーメッセージ
- `clearError()` - エラー解除処理

**仕様:**
- タイムアウト値は `NEXT_PUBLIC_ERROR_NOTIFICATION_TIMEOUT`、未設定時は `5000ms` を使用する。
- `errorMessage` が変化したときにタイマーを張り直す。
- close ボタン押下時は即時 `clearError()` を実行する。
- 現行実装では `style jsx` によるローカルスタイルで見た目を制御する。
