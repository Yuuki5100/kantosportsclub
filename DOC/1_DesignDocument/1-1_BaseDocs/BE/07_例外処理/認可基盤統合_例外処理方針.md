# 認可基盤統合_例外処理方針

## 1. 目的
認可判定で発生する拒否系例外と応答仕様を統一する。

## 2. 適用範囲
- `RequirePermissionInterceptor`
- `RolePermissionChecker`
- `PermissionChecker`
- `AuthorizationInterceptor`（legacy）

## 3. 例外・応答ルール

| ケース | 例外/応答 | 文言 |
| --- | --- | --- |
| 権限不足 | `AccessDeniedException` / HTTP 403 | `BackendMessageCatalog.MSG_ACCESS_DENIED` |
| `@RequirePermission` 未設定 | `AccessDeniedException` / HTTP 403 | `BackendMessageCatalog.EX_REQUIRE_PERMISSION_MISSING` |
| legacy interceptor拒否 | `response.sendError(403)` | `BackendMessageCatalog.MSG_ACCESS_DENIED` |

## 4. ログルール
- スキップログ: `LOG_REQUIRE_PERMISSION_SKIP`
- 拒否ログ: `LOG_REQUIRE_PERMISSION_ACCESS_DENIED`
- 権限定義ロードログ: `LOG_ENDPOINT_CONFIG_*`

## 5. 注意事項
- 例外文言は共通カタログを使用し、独自文字列を直書きしない
- エラー時に内部実装情報（SQL/stacktrace）をレスポンスへ露出しない
