# 帳票出力_例外設計書

## 1. 目的
帳票出力および非同期ジョブ関連の例外方針を統一する。

## 2. 方針
- 帳票生成例外は `ReportGenerationException` を基本とする。
- APIレスポンスは共通ハンドリング方針に従う。
- ログ文言・例外文言は `BackendMessageCatalog` を使用する。

## 3. 主要エラーケース

| ケース | 発生箇所 | 挙動 |
| --- | --- | --- |
| 帳票種別不正 | `/report/export/{type}` `/report/download/{type}` | `400 Bad Request` |
| 非同期起動パラメータ不正 | `/report/job` | `E4003` |
| 非同期実行中の生成失敗 | `ReportPollingRunner` | `async_job_execution=FAILED` |
| 成果物再取得失敗 | `/import/download` | `500 Internal Server Error` |
| 期限切れ掃除中の削除失敗 | `AsyncJobCleanupService` | WARNログ、処理継続 |

## 4. 非同期失敗時ルール
- 失敗時は可能な限り成果物を削除する。
- 状態は `FAILED` に更新し `error_message` を保存する。
- ポーリング利用者は `FAILED` を検知して再実行判断する。

## 5. 監査性
- 状態遷移（`PENDING/RUNNING/COMPLETED/FAILED/EXPIRED`）と時刻をDB保存する。
- 生成・削除エラーはログで追跡可能にする。
