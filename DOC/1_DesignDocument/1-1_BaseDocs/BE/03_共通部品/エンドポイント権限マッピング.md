# エンドポイント権限マッピング 共通部品設計書

## 1. 目的
`endpoint_authority_mapping` 正本を、認可判定で扱いやすい内部マップへ変換する仕様を定義する。

## 2. 変換対象
- 入力: `endpoint_authority_mapping`
- 出力: `Map<String, EndpointPermissions>`

## 3. 変換ルール
- キー: `METHOD + " " + URL`（METHODは大文字）
- `custom`: `menuFunctionId` を文字列化してキー化
- 値: `requiredLevel`（重複時最大値）
- `defaultLevel`: 同一エンドポイントの `requiredLevel` 最大値

## 4. 変換実装
- 実装クラス: `PermissionConfigProviderImpl`
- 更新契機:
  - 起動時 `@PostConstruct`
  - 定期再読込 `@Scheduled`
  - 明示呼び出し `refresh()`

## 5. 利用側
- `PermissionChecker`
- `CustomPermissionEvaluator`（経由）

## 6. 注意事項
- 未定義エンドポイントは `PermissionChecker` では拒否
- Method Security補助経路の未定義挙動は別設定で制御
