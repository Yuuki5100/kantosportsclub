# ファイル取込 共通部品設計書
---

## 1. 目的
ファイル取込に関する共通部品を実装に合わせて整理する。

## 2. 適用範囲
- 対象モジュール: `servercommon`
- 対象層: Component

## 3. モジュール概要
### 3-1. 役割
アップロード/検証/保存/履歴保存の共通処理を提供する。

### 3-2. 分類
Component

## 4. 構成要素

| 種別 | 名称 | 役割 |
|---|---|---|
| Service | `ImportJobExecutor` | 取込処理の実行 |
| Service | `StorageService` | ストレージ抽象化 |
| Service | `S3StorageService` | S3実装 |
| Service | `LocalFileStorageService` | ローカル実装 |
| Utility | `FileSaver` | `StorageService.upload` |
| Utility | `FileNameResolver` | 日付付きファイル名 |
| Utility | `FileTypeResolver` | 拡張子判定 |
| Validator | `FileValidatorDispatcher` | Validator呼び分け |
| Repository | `JobStatusRepository` | 取込履歴 |
| Model | `JobStatus` | 取込結果 |
| Mapper | `TemplateGetMapper` | YAML取得 |

## 5. 入出力仕様
### 5-1. `ImportJobExecutor`

| 入力 | 出力 | 概要 |
|---|---|---|
| `MultipartFile` または `File` | `JobStatus` 保存 | 取込・検証・保存 |

### 5-2. `StorageService`

| 入力 | 出力 |
|---|---|
| `upload(String, InputStream)` | なし |
| `getFileByPath(String)` | `File` |
| `generatePresignedUrl(String)` | `URL` |

## 6. 処理フロー
1. `FileNameResolver` で日付付与
2. `FileSaver` で保存
3. `FileValidatorDispatcher` で検証
4. `UserWriter` による書き込み
5. `JobStatus` を保存

## 7. 設計方針
- ストレージ差異は `StorageService` に吸収
- ファイル名は日付サフィックスで一意化

## 8. 依存関係
- `JobStatusRepository`
- `FileValidatorDispatcher`
- `StorageService`

## 9. 利用箇所
- `BE/batchserver/src/main/java/com/example/batchserver/scheduler/FileImportScheduler.java`
- `BE/appserver/src/main/java/com/example/appserver/controller/FileImportController.java`
- `BE/appserver/src/main/java/com/example/appserver/runner/FileImportRunner.java`

## 10. 実装との整合
- `FileImportScheduler` で `ImportJobExecutor.execute` がコメントアウト

## 11. 制約・注意事項
- `FileTypeResolver.resolveJobType` は CSV/Excel のみ対応

## 12. テスト観点
- 正常取込と履歴保存
- 例外時の `JobStatus` 生成

## 13. 要確認事項
- `ImportJobExecutor` の実運用経路
