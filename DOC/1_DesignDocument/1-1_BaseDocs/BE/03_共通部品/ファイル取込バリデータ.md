# ファイル検証（テンプレート型）共通部品設計書
---

## 1. 目的
テンプレート定義に基づくバリデーションを共通化する。

## 2. 適用範囲
- 対象モジュール: `servercommon`
- 対象層: Validator
- 対象クラス: `GenericValidator` と各種 `ValidationRule`

## 3. モジュール概要
### 3-1. 役割
CSV/Excelの各行・各列に対し、テンプレート定義のルールで検証する。

### 3-2. 分類
Validator

## 4. 構成要素

| 種別 | 名称 | 役割 |
|---|---|---|
| Validator | `GenericValidator` | 検証統括 |
| Validator | `ValidationRuleRegistry` | ルールの既定セット |
| Validator | `RequiredRule` | 必須チェック |
| Validator | `EnumRule` | 列挙値チェック |
| Validator | `PatternRule` | 正規表現チェック |
| Validator | `MaxLengthRule` | 最大長チェック |
| Validator | `HeaderFormatValidator` | ヘッダ整合性チェック |
| Model | `ValidationResult` | 検証結果 |

## 5. 入出力仕様
### 5-1. メソッド

| メソッド | 入力 | 出力 | 概要 |
|---|---|---|---|
| `GenericValidator.validate(List<Map<String,String>>, TemplateSchema)` | records, schema | `List<ValidationResult>` | v1形式検証 |
| `GenericValidator.validate(List<Map<String,String>>, List<ColumnSchema>)` | records, columns | `List<ValidationResult>` | v2形式検証 |

### 5-2. Dynamic Required

| インタフェース | 入力 | 出力 |
|---|---|---|
| `RowBasedRequiredRule.isRequired(Map<String,String>)` | row | `boolean` |
| `ContextBasedRequiredRule.init(List<Map<String,String>>)` | allRows | なし |
| `ContextBasedRequiredRule.isRequired(Map<String,String>)` | row | `boolean` |

## 6. 処理フロー
1. `HeaderFormatValidator` を実行
2. 各行を走査し `RequiredRule` を適用
3. 既定ルール（Enum/Pattern/MaxLength）を適用
4. `ValidationResult` を返却

## 7. 設計方針
- 必須判定は `dynamicRequiredRule` を優先
- 空行は `FileExecUtils.fileEmptyRowsSkip` で除外

## 8. 依存関係
- `ErrorCodeService`
- `TemplateSchema` / `ColumnSchema`

## 9. 利用箇所
- `GenericFileImporter` 内部で利用
- `FileImportService` などで間接利用

## 10. 実装との整合
- ルールセットは `ValidationRuleRegistry.getDefaultRules` と一致
- `ValidationResult` の構造は実装と一致

## 11. 制約・注意事項
- `dynamicRequiredRule` はクラス名指定が必須
- ルールの追加は `ValidationRuleRegistry` 改修が必要

## 12. テスト観点
- ルール別の正常/異常
- dynamicRequiredRule の優先順位
- ヘッダ不整合の検出

## 13. 要確認事項
- 追加ルールをどの層で拡張するか
