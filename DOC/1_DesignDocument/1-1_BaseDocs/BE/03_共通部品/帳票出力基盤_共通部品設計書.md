# 帳票出力基盤 共通部品設計書

## 1. 目的
帳票生成と非同期配布に関わる共通部品の責務を定義する。

## 2. 適用範囲
- 対象モジュール: `servercommon`（生成・保存） / `appserver`（起動・ポーリング）

## 3. 構成要素

| 種別 | 名称 | 役割 |
| --- | --- | --- |
| Service | `ReportServiceImpl` | 単一帳票生成（PDF/Excel） |
| Service | `BulkReportExportServiceImpl` | 一括帳票生成 |
| Component | `ReportDefinitionLoader` | 帳票定義ロード |
| Utility | `PdfGenerator` / `ExcelGenerator` | 帳票生成 |
| Service | `StorageService` | テンプレート・成果物I/O |
| Service | `AsyncJobStatusService` | 非同期ジョブ状態永続化 |
| Service | `AsyncJobArtifactService` | 成果物保存/再取得/削除 |

## 4. 主要I/F

### 4-1. 帳票生成
- `generateReportExcel(Long reportId): InputStream`
- `generateReportPDF(Long reportId): InputStream`
- `generateReportDownloadUrl(Long reportId, String fileName): String`

### 4-2. 非同期ジョブ
- `registerPending(jobName, jobType, expiresAt)`
- `markRunning(jobName, jobType, expiresAt)`
- `markCompleted(jobName, artifactPath, artifactMimeType, endedAt)`
- `markFailed(jobName, errorMessage, endedAt)`
- `markExpired(jobName)`

## 5. 設計方針
- 状態は `async_job_execution` へ保存し、メモリキャッシュを正としない。
- 成果物は `StorageService` 配下に保存し `artifact_path` で再取得する。
- 互換性維持のため `job_status` は履歴系APIで継続利用する。

## 6. 制約
- `error_message` は1000文字で切り詰める。
- `job_name` は一意制約で重複登録を防止する。
