# 共通エラーハンドリング・メッセージカタログ設計書

## 1. 目的
バックエンド全体で利用するエラー応答・ログ文言・例外文言を一元化し、再利用性と保守性を高める。

本設計書は、以下の共通モジュールの利用ルールを定義する。
- `BackendMessageCatalog`
- `BackendMessageResolver`
- `GlobalExceptionHandler`（連携先）

## 2. 適用範囲
- 対象モジュール: `servercommon`, `appserver`, `batchserver`, `gateway`, `syncconnector`
- 対象レイヤ: Controller / Service / Batch / 共通部品
- 対象メッセージ:
  - APIエラーメッセージ（`ApiResponse.error`）
  - ログ文言（`log.info/warn/error/...`）
  - 例外メッセージ（`throw new ...` / `CustomException`）

## 3. 構成要素
| 種別 | クラス | 役割 |
| --- | --- | --- |
| Catalog | `com.example.servercommon.message.BackendMessageCatalog` | エラーコード・ログ文言・例外テンプレートの定数を集約 |
| Resolver | `com.example.servercommon.message.BackendMessageResolver` | `ErrorCodeService` を利用し、ロケール付きでエラーメッセージを解決 |
| Handler | `com.example.servercommon.exception.GlobalExceptionHandler` | 例外を捕捉し `ApiResponse.error` を返却 |

## 4. 設計方針
## 4-1. エラーコードの一元化
- エラーコードは `BackendMessageCatalog.CODE_*` を利用する。
- ハードコード（例: `"E4001"` の直接記述）は新規実装で禁止。

## 4-2. エラーメッセージの解決
- API返却メッセージは `BackendMessageResolver.resolveError(...)` を利用する。
- 直接文字列（例: `"Unauthorized"`）を `ApiResponse.error` に渡さない。

## 4-3. ログ文言の統一
- ログ文言は `BackendMessageCatalog.LOG_*` を利用する。
- `System.out.println` / `System.err.println` は禁止。

## 4-4. 例外文言の統一
- 例外メッセージは `BackendMessageCatalog.EX_*` を優先利用する。
- フォーマットが必要な場合は `BackendMessageCatalog.format(...)` を使用する。

## 5. 利用ルール（必須）
1. `ApiResponse.error(code, message)` の `code` は `BackendMessageCatalog.CODE_*` を使う。
2. `message` は `BackendMessageResolver` または `BackendMessageCatalog` から取得する。
3. ログ文言は `BackendMessageCatalog.LOG_*` を使う。
4. 例外生成時の文言は `BackendMessageCatalog.EX_*` を使う。
5. 新規メッセージ追加時は、まず `BackendMessageCatalog` に定義する。

## 6. 実装例
```java
String message = messageResolver.resolveError(BackendMessageCatalog.CODE_E401, locale);
return ResponseEntity.status(401)
        .body(ApiResponse.error(BackendMessageCatalog.CODE_E401, message));
```

```java
log.error(BackendMessageCatalog.LOG_FILE_UPLOAD_FAILED, e);
```

```java
throw new RuntimeException(
        BackendMessageCatalog.format(BackendMessageCatalog.EX_TOKEN_REFRESH_FAILED, e.getMessage()), e);
```

## 7. 追加・変更手順
1. `BackendMessageCatalog` にコード/文言を追加する。
2. エラーコードを追加する場合は `error_codes`（DB/SQL）へ追記する。
3. 利用側（Controller/Service/Handler）を定数参照に置換する。
4. 再利用ルール文書（`BE/reuse`）へ必要事項を追記する。

## 8. 既知の課題
- 既存コードにはログ直書き・例外文言直書きが残存しているため、段階移行が必要。
- CIレベルでの静的チェック（直書き禁止ルール）は今後追加予定。
