# 汎用ファイル取込エンジン 共通部品設計書
---

## 1. 目的
YAMLテンプレートに基づくCSV/Excelの汎用取込処理を共通化する。

## 2. 適用範囲
- 対象モジュール: `servercommon`
- 対象層: Component
- 利用対象: ファイル取込の共通実装（appserver/batchserver）

## 3. モジュール概要
### 3-1. 役割
テンプレート定義の読み込み、レコード読取、エンティティ変換、リポジトリ保存を一括で実行する。

### 3-2. 分類
Component

## 4. 構成要素

| 種別 | 名称 | 役割 |
|---|---|---|
| Component | `GenericFileImporter` | 取込全体のオーケストレーション |
| Component | `TemplateSchemaLoader` | YAMLテンプレートの読み込み |
| Component | `FileRecordReaderDispatcher` | CSV/Excel読取の分岐 |
| Component | `CsvRecordReader` | CSVのレコード読取 |
| Component | `ExcelRecordReader` | Excelのレコード読取 |
| Component | `GenericEntityMapper` | Map→Entity 変換 |
| Component | `RepositoryResolver` | Repository解決 |
| Model | `TemplateSchema` | テンプレート定義 |
| Model | `TemplateSheetSchema` | シート定義 |
| Model | `ColumnSchema` | カラム定義 |

## 5. 入出力仕様
### 5-1. `GenericFileImporter`

| メソッド | 入力 | 出力 | 概要 |
|---|---|---|---|
| `importFiles(Map<String,TemplateSchema>, String, String, InputStream)` | テンプレートMap, templateId, filename, InputStream | `List<ValidationResult>` | 読取・検証・保存を実行 |

### 5-2. テンプレートロード

| メソッド | 入力 | 出力 | 概要 |
|---|---|---|---|
| `TemplateSchemaLoader.load(String)` | `templateId` | `TemplateSchema` | `template.schema.basePath` から読み込み |
| `TemplateSchemaLoader.loadFromPath(String)` | path | `TemplateSchema` | 任意パス読み込み |

### 5-3. レコード読取

| メソッド | 入力 | 出力 | 概要 |
|---|---|---|---|
| `FileRecordReaderDispatcher.read(String, InputStream)` | filename, stream | `List<Map<String,String>>` | CSV/Excelの分岐 |
| `FileRecordReaderDispatcher.readSheet(String, String, InputStream)` | filename, sheetName, stream | `List<Map<String,String>>` | Excelシート指定 |

### 5-4. 変換/保存

| メソッド | 入力 | 出力 | 概要 |
|---|---|---|---|
| `GenericEntityMapper.map(List<Map<String,String>>, List<ColumnSchema>, Class<T>)` | records, columns, class | `List<T>` | エンティティ変換 |
| `RepositoryResolver.resolveRepository(List<ColumnSchema>)` | columns | `Object` | Repository取得 |

## 6. 処理フロー
1. `TemplateSchemaLoader.load` でテンプレートを取得
2. `FileRecordReaderDispatcher` でCSV/Excelを読取
3. `SkipRule` があればレコードを除外
4. `GenericValidator` で検証
5. `GenericEntityMapper` でエンティティ変換
6. `RepositoryResolver` で `saveAll` を実行

## 7. 設計方針
- `InputStream` は一度バイト配列化して再利用可能にする
- v1は単一シート、v2は複数シートを処理
- Repositoryは `ColumnSchema.repository` によって解決

## 8. 依存関係
- `jackson-dataformat-yaml`
- `Apache POI`
- `Apache Commons CSV`
- `Spring ApplicationContext`

## 9. 利用箇所
- `BE/appserver/src/main/java/com/example/appserver/service/FileImportService.java`
- `BE/appserver/src/main/java/com/example/appserver/runner/FileImportRunner.java`
- `BE/batchserver/src/main/java/com/example/batchserver/tasklet/FileValidationTasklet.java`

## 10. 実装との整合
- 実装に存在する構成要素に合わせて整理
- 取込前後のログ出力は実装通り

## 11. 制約・注意事項
- `TemplateSchema.isMultiSheet()` が `true` の場合のみシート指定読取を使用
- `RepositoryResolver` は `ColumnSchema.repository` が空の場合は例外
