# 非同期ジョブ状態永続化_共通部品設計書

## 1. 目的
非同期ジョブの状態・成果物を永続化し、再起動後も追跡可能にする。

## 2. 適用範囲
- `BE/servercommon`
  - `model.AsyncJobExecution`
  - `repository.AsyncJobExecutionRepository`
  - `service.AsyncJobStatusService`
  - `service.AsyncJobArtifactService`
- `BE/appserver`
  - `service.AsyncJobCleanupService`

## 3. 状態モデル
- Enum: `AsyncJobExecutionStatus`
  - `PENDING`, `RUNNING`, `COMPLETED`, `FAILED`, `EXPIRED`

## 4. Repository
- `findByJobName(String jobName)`
- `findExpiredTargets(LocalDateTime now, Pageable pageable)`
  - 対象: `COMPLETED` / `FAILED` かつ `expires_at <= now`

## 5. Service責務

### 5-1. `AsyncJobStatusService`
- ジョブ登録・状態遷移を `@Transactional` で提供。
- `markFailed` 時は `error_message` を保存。
- `markExpired` 時は成果物参照情報をクリア。

### 5-2. `AsyncJobArtifactService`
- `save(artifactPath, InputStream)`
- `open(artifactPath)`
- `generateDownloadUrl(artifactPath)`
- `deleteQuietly(artifactPath)`

### 5-3. `AsyncJobCleanupService`
- `@Scheduled(fixedDelayString = "${async.job.cleanup-fixed-delay-ms:600000}")`
- 期限切れ対象を `cleanup-batch-size` 件ずつ処理。

## 6. トランザクション・整合方針
- DB更新とストレージ更新を分散Txにはしない。
- 失敗時は成果物削除を試行し、状態を `FAILED` に収束させる。

## 7. 関連設定
- `async.job.status-ttl-minutes`
- `async.job.cleanup-fixed-delay-ms`
- `async.job.cleanup-batch-size`
- `async.job.artifact-prefix`
