# 一括帳票出力機能設計書

## 1. 目的
複数帳票を一括で出力し、単一ファイルとして提供する。

## 2. ユースケース
- 複数の帳票をまとめてPDFで取得する。
- 複数の帳票を1ファイルのExcel（複数シート）として取得する。

## 3. API仕様

### 3-1. API一覧
| No | Method | Path | 内容 |
|----|--------|------|------|
| 1 | POST | `/report/export/bulk` | 一括帳票出力 |

### 3-2. リクエスト
- `BulkExportRequest`

| 項目 | 型 | 必須 | 説明 |
|------|----|------|------|
| reportIds | List<Long> | 要確認 | 帳票IDの配列 |
| outputFormat | String | 要確認 | `pdf` / `excel` |
| fileNamePrefix | String | 任意 | ファイル名接頭語 |

### 3-3. レスポンス
- `ApiResponse<String>`
- 返却値は保存先の署名付きURL

## 4. 処理フロー
1. `BulkReportExportController` がリクエストを受信する。
2. `BulkReportExportServiceImpl` が帳票定義を取得し、PDF/Excel を結合する。
3. 結果をストレージへ保存し、署名付きURLを返却する。

## 5. モジュール構成
| 種別 | 実装名 | 役割 |
|------|--------|------|
| Controller | `BulkReportExportController` | API受信 |
| Service | `BulkReportExportService` / `BulkReportExportServiceImpl` | 一括出力 |
| Request DTO | `BulkExportRequest` | リクエスト |
| Storage | `StorageService` | 署名付きURL生成 |

## 6. 実装との整合
- PDFは `PDFMergerUtility` で結合している。
- Excelは `XSSFWorkbook` で複数シートに統合している。
- 返却値は Base64 ではなく署名付きURLである。
