# 一括帳票出力機能（参照用サマリ）

## 1. 目的
一括帳票出力機能の要点を簡潔に整理する。詳細は `一括帳票出力機能設計書.md` を正本とする。

## 2. ユースケース
- 複数帳票をまとめてPDFで取得する。
- 複数帳票を1ファイルのExcel（複数シート）として取得する。

## 3. API仕様

### 3-1. API一覧
| No | Method | Path | 内容 |
|----|--------|------|------|
| 1 | POST | `/report/export/bulk` | 一括帳票出力 |

### 3-2. リクエスト
- `BulkExportRequest`

| 項目 | 型 | 必須 | 説明 |
|------|----|------|------|
| reportIds | List<Long> | 要確認 | 帳票IDの配列 |
| outputFormat | String | 要確認 | `pdf` / `excel` |
| fileNamePrefix | String | 任意 | ファイル名接頭語 |

### 3-3. レスポンス
- `ApiResponse<String>`
- 返却値は保存先の署名付きURL

## 4. 処理フロー
1. `BulkReportExportController` がリクエストを受信する。
2. `BulkReportExportServiceImpl` が帳票を結合し、ストレージへ保存する。
3. 署名付きURLを返却する。

## 5. モジュール構成
| 種別 | 実装名 | 役割 |
|------|--------|------|
| Controller | `BulkReportExportController` | API受信 |
| Service | `BulkReportExportService` | 一括出力 |
| Request DTO | `BulkExportRequest` | リクエスト |
| Storage | `StorageService` | 署名付きURL生成 |

## 6. 実装との整合
- 実装ではBase64返却は行っていない。
- 返却値は `StorageService.generatePresignedUrl` によるURLである。
