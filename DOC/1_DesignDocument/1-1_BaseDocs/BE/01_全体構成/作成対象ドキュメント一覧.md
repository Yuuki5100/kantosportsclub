## 3. 実装確認結果

### 3-1. `syncconnector` の現行実装

- 独立した Spring Boot モジュールとして存在する
- DB / JPA / Flyway / Batch 自動構成を除外した軽量構成になっている
- 受信時は `SyncSignatureVerificationConfig` が `ContentCachingRequestWrapper` 用 Filter と `SyncSignatureVerificationInterceptor` を登録する
- 送信時は `SignedRestTemplateConfig` が `SignedRestTemplate` を提供し、HMAC 署名付き POST を実行できる
- 署名アルゴリズムは `HmacSigner` に集約されている
- 署名検証失敗は `SignatureVerificationException` から `GlobalExceptionHandler` で 401 に変換される
- Docker でも `syncconnector` サービスとして任意起動対象に含まれる
- `appserver` 側は `sync.outbox.use` により送信基盤を有効/無効化できる
- `sync.outbox.use=false` の構成では `syncconnector` 依存を持たない運用を許容する

### 3-2. 粒度評価

- 既存の `GateWay.md` や `認証認可_基盤設計書.md` と同じく、単独モジュールかつ基盤責務を持つ
- 単なる個別 API ではなく、外部連携の受信検証と送信署名の双方を担うため、`02_基盤アーキテクチャ` に相当する
- 処理経路も送信と受信で明確に分かれるため、`03_処理方式` に相当する文書化対象でもある
