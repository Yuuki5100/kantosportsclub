# 帳票出力_全体像設計書

## 目的
帳票出力の全体像を整理し、同期/非同期の責務分担を明確化する。

## 全体アーキテクチャ図

```mermaid
graph LR;
    User["ユーザー"] -->|帳票出力リクエスト| FE["フロントエンド"]
    FE -->|API呼び出し| App["appserver"]

    App -->|帳票生成| ReportService["ReportService"]
    ReportService -->|テンプレート/成果物I/O| Storage["StorageService"]
    ReportService -->|データ取得| DB["DB"]

    App -->|非同期状態更新| AsyncState["async_job_execution"]
    App -->|履歴互換参照| JobStatus["job_status"]

    Storage --> S3["S3（storage.type=s3）"]
    Storage --> Local["Local（storage.type=local）"]
```

## 全体シーケンス図

```mermaid
sequenceDiagram
    participant User as ユーザー
    participant FE as フロントエンド
    participant App as appserver
    participant Judge as ReportTypeJudge
    participant Runner as ReportPollingRunner
    participant State as AsyncJobStatusService
    participant Artifact as AsyncJobArtifactService

    User->>FE: 帳票出力要求
    FE->>App: POST /report/job
    App->>Judge: 起動判定
    Judge->>State: registerPending(jobName)
    Judge->>Runner: 非同期起動
    App-->>FE: jobName

    FE->>App: GET /report/polling/{jobName}
    App->>State: findByJobName(jobName)
    State-->>App: status + artifactPath
    App-->>FE: status + url

    Runner->>State: markRunning/markCompleted/markFailed
    Runner->>Artifact: save/delete
```

## 注記
- 非同期起動は実装済み（`ReportTypeJudge` から `ReportPollingRunner` を呼び出す）。
- 状態管理は `JobStatusCache` ではなく `async_job_execution` を主系とする。
- `job_status` は互換用途で継続参照する。
