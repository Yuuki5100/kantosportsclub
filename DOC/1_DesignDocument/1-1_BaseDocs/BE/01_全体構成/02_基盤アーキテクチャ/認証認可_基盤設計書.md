# 認証認可_基盤設計書

## 1. 目的
認証・認可の実装経路を一本化し、権限判定の正本と互換経路を明確化する。

## 2. 適用範囲
- `BE/appserver` の security / interceptor / permission / config
- 認証方式切替（`auth.type`, `security.auth.mode`）
- 認可方式（`@RequirePermission` / Method Security / 旧Interceptors）

## 3. 正本と判定経路

### 3-1. 権限定義の正本
- エンドポイント要求定義の正本: `endpoint_authority_mapping`
- ロール保有権限の正本: `role_permission`
- ロールID定義: `UserRole`（`SYSTEM_ADMIN=1`, `EDITOR=2`, `VIEWER=3`, `CUSTOM=4`）

### 3-2. 標準経路（推奨）
1. Controller に `@RequirePermission(permissionId, statusLevelId)` を付与
2. `RequirePermissionInterceptor` が注釈値を取得
3. `RolePermissionChecker` が `role_permission` を参照
4. `actualLevel >= statusLevelId` で許可、満たさなければ 403

### 3-3. 補助経路（Method Security）
- `@PreAuthorize` から `PermissionChecker` / `CustomPermissionEvaluator` を利用可能
- `CustomPermissionEvaluator` は `PermissionChecker` に委譲し、
  `security.permission.method-security-*` で未定義時挙動を制御する

### 3-4. 旧経路（互換）
- `AuthorizationInterceptor` は `security.authorization.legacy-interceptor-enabled=true` のときのみ有効
- `EndpointPermissionConfig`（in-memory定義）は `security.permission.legacy-endpoint-config-enabled=true` のときのみ有効

## 4. 設定値

| キー | 既定値 | 用途 |
| --- | --- | --- |
| `security.authorization.legacy-interceptor-enabled` | `false` | 旧AuthorizationInterceptor経路の有効化 |
| `security.permission.legacy-endpoint-config-enabled` | `false` | in-memory `EndpointPermissionConfig` の有効化 |
| `security.permission.cache-refresh-fixed-delay-ms` | `600000` | 権限定義キャッシュ再読込間隔(ms) |
| `security.permission.method-security-allow-if-unmapped` | `true` | Method Security補助経路で未定義を許可するか |
| `security.permission.method-security-default-method` | `GET` | Method SecurityでHTTPメソッド省略時の既定値 |

## 5. 全体フロー

### 5-1. 認証
- `/api/auth/*` は `AuthService` が `AuthFlow`（internal/gbiz）を切替
- `security.auth.mode` により Session/JWT を切替

### 5-2. 認可
- 標準: `RequirePermissionInterceptor -> RolePermissionChecker`
- 補助: `@PreAuthorize -> PermissionChecker / CustomPermissionEvaluator`
- 互換: `AuthorizationInterceptor`（設定有効時のみ）

## 6. 主要コンポーネント
- `RequirePermissionInterceptor`
- `RolePermissionChecker`
- `PermissionChecker`
- `CustomPermissionEvaluator`
- `PermissionConfigProviderImpl`
- `AuthorizationModeProperties`, `PermissionProperties`

## 7. 互換性方針
- 既存の `@RequirePermission` API は仕様変更なし
- Method Security は既存式を維持しつつ、未定義時挙動を設定で制御
- 旧経路はデフォルト無効（必要時のみ明示有効化）
