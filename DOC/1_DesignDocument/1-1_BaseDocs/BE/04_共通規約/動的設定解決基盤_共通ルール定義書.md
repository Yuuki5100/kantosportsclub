# 動的設定解決基盤_共通ルール定義書

## 1. 目的
動的設定の取得・更新・履歴管理に関する実装揺れを防ぎ、運用時の追跡性を確保する。

## 2. 適用範囲
- `appserver`
- `servercommon`
- 対象処理: `SystemSettingResolver`, `SystemSettingService`, `SystemSettingHistoryService`

## 3. 共通ルール

### 3-1. 正本と参照経路
- 設定値の正本は `system_setting` とする。
- 業務可変設定の取得は `SystemSettingResolver` 経由に統一し、Repository直参照を新規追加しない。

### 3-2. キー管理
- 設定キーは `SystemSettingKeys` に集約する。
- 文字列リテラルでのキー直書きを禁止する。
- 既存互換キー（`noticeDisplayLimit`）は廃止まで `SystemSettingKeys` で管理する。

### 3-3. 型変換
- 型変換は `SystemSettingValueConverter` に集約する。
- `Boolean` は `true/false` のみ許容する。
- 変換失敗時は `IllegalArgumentException` を送出し、握りつぶさない。

### 3-4. キャッシュ
- キャッシュ制御値は `system.setting.cache.*` に外出しする。
- 設定更新時は `evictAll()` を必須とする。
- 高頻度更新値を追加する場合は、キャッシュ有効範囲を個別に見直す。

### 3-5. 履歴管理
- 設定更新時は `system_setting_history` に差分のみ保存する。
- `updated_by` と `updated_date_and_time` の記録を必須とする。

### 3-6. ログ・例外
- ログ文言は `BackendMessageCatalog.LOG_SYSTEM_SETTING_*` を利用する。
- 例外文言は `BackendMessageCatalog.EX_SYSTEM_SETTING_*` を利用する。

## 4. NGパターン
- `@Value` で業務可変設定を直接取得する。
- 設定更新後にキャッシュ無効化を行わない。
- 設定更新時に履歴を残さない。
