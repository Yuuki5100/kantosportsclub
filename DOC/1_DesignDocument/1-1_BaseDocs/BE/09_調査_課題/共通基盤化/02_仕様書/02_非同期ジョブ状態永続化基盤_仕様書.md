# 非同期ジョブ状態永続化基盤_仕様書

## 1. 目的

非同期ジョブ状態と生成物を永続化し、再起動耐性を持つジョブ参照基盤を提供する。

## 2. 適用範囲

- 帳票出力
- ダウンロード準備
- 今後追加される非同期ジョブ

## 3. 構成要素

| 区分 | 名称 | 役割 |
| --- | --- | --- |
| Model | `AsyncJobExecution` | ジョブ実行状態 |
| Repository | `AsyncJobExecutionRepository` | 状態永続化 |
| Service | `AsyncJobStatusService` | 状態更新 |
| Service | `AsyncJobArtifactService` | 生成物保管 |
| API | `downloadReady` / `download` | 状態参照・取得 |

## 4. 状態仕様

| 状態 | 意味 |
| --- | --- |
| `PENDING` | 受付済み |
| `RUNNING` | 実行中 |
| `COMPLETED` | 完了 |
| `FAILED` | 失敗 |
| `EXPIRED` | 取得期限切れ |

## 5. 保持項目

- `job_name`
- `job_type`
- `status`
- `artifact_path`
- `artifact_mime_type`
- `error_message`
- `started_at`
- `ended_at`
- `expires_at`

## 6. 処理フロー

1. ジョブ受付時に `PENDING` を登録する
2. 実行開始時に `RUNNING` に更新する
3. 生成物作成後にストレージへ保存する
4. 成功時に `COMPLETED` と `artifact_path` を更新する
5. 失敗時に `FAILED` と `error_message` を更新する
6. ダウンロード API は `artifact_path` から取得する

## 7. 生成物仕様

- 実体は `StorageService` 管理とする
- API はストレージの実体を返す
- 期限切れ後は削除対象とする

## 8. 互換方針

- 既存の `jobName` ベース API は維持する
- 実装内部のみメモリキャッシュから永続化へ置き換える

## 9. クリーンアップ仕様

- `expires_at` 超過分を定期削除する
- 削除時にストレージ上の成果物も削除する

## 10. テスト観点

- 状態遷移
- 再起動後参照
- ダウンロード成功
- 期限切れ削除
