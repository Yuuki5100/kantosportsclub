# 動的設定解決基盤_仕様書

## 1. 目的

システム設定を DB から共通的に解決する仕組みを提供し、コード修正なしで運用変更可能な設定基盤を作る。

## 2. 適用範囲

- `SystemSetting`
- 設定取得サービス
- 各モジュールからの参照

## 3. 構成要素

| 区分 | 名称 | 役割 |
| --- | --- | --- |
| Interface | `SystemSettingResolver` | 設定取得 API |
| Service | `SystemSettingService` | 更新・管理 |
| Utility | `SystemSettingValueConverter` | 型変換 |
| Cache | `SystemSettingCache` | 参照高速化 |
| Repository | `SystemSettingRepository` | 永続化 |
| Optional | `SystemSettingHistoryRepository` | 履歴参照 |

## 4. 取得仕様

- `getString(key)`
- `getInt(key)`
- `getBoolean(key)`
- `getDuration(key)`
- `getOrDefault(key, defaultValue)`

## 5. データモデル方針

- 現行単一レコード方式から、将来的には key-value 方式へ拡張可能にする
- 移行完了までは互換変換層を設ける

## 6. キャッシュ仕様

- 設定取得結果をキャッシュする
- 更新時に無効化する
- TTL は設定値化する

## 7. 変更履歴仕様

- 変更前値
- 変更後値
- 更新者
- 更新時刻

## 8. 例外仕様

- 設定未存在時は例外またはデフォルト返却
- 型変換失敗時は設定エラー例外

## 9. 利用想定

- パスワードポリシー
- 表示件数
- 通知上限
- 将来の運用フラグ

## 10. テスト観点

- 正常取得
- デフォルト取得
- 型変換失敗
- 更新後の反映
