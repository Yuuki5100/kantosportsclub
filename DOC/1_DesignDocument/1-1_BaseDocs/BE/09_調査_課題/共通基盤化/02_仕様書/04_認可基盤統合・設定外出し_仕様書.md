# 認可基盤統合・設定外出し_仕様書

## 1. 目的

エンドポイント権限判定を共通基盤として統合し、設定の正本を明確化する。

## 2. 適用範囲

- `PermissionChecker`
- `CustomPermissionEvaluator`
- `RequirePermission`
- `PermissionConfigProvider`
- 権限マスタ

## 3. 基本方針

- 権限制御の正本は DB とする
- 判定は `@RequirePermission` 経由を標準とする
- Method Security は補助用途に限定する

## 4. 構成要素

| 区分 | 名称 | 役割 |
| --- | --- | --- |
| Annotation | `RequirePermission` | エンドポイント要求権限 |
| Service | `PermissionChecker` | 実判定 |
| Provider | `PermissionConfigProvider` | 設定提供 |
| Cache | `PermissionCache` | 権限キャッシュ |
| Repository | `EndpointAuthorityMappingRepository` | DB 正本取得 |

## 5. 判定仕様

1. リクエスト対象の `METHOD URL` を解決する
2. 要求権限を取得する
3. システム管理者は常時許可する
4. カスタム権限は `permissionId` ごとに判定する
5. 通常ロールはロールレベルで判定する

## 6. 設定仕様

- 権限設定変更時はキャッシュを破棄する
- 起動時に全件ロードする
- 一定期間ごとの再読込はオプションとする

## 7. 互換仕様

- 既存の `@RequirePermission` は維持する
- `EndpointPermissionConfig` は移行期間の互換用途に限定する

## 8. 運用仕様

- 権限未設定エンドポイントは拒否または明示許可ポリシーを選択する
- 判定ログを debug で出力できるようにする

## 9. テスト観点

- 管理者ロール
- 通常ロール
- カスタム権限
- 設定更新後の再読込
