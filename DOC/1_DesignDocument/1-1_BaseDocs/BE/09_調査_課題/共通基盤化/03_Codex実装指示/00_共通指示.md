# Codex 共通指示

以下は、各機能を別 Codex へ依頼する際に必ず前提とする共通指示です。

## 1. あなたの役割

- あなたはこの機能の実装担当です。
- 他の Codex も並行して別機能を実装している前提で作業してください。
- 自分の担当範囲外の既存実装は原則変更しないでください。
- 同じフォルダに他者の変更が入る可能性があるため、他者の変更を巻き戻さないでください。

## 3. 再利用ルールの必須遵守事項

- 既存共通部品を使わずに独自実装しないこと
- 既存機能を再実装しないこと
- 外部 API を直叩きせず、既存の共通連携部品を使うこと
- エラーコード、ログ文言、例外文言を直書きしないこと
- `ApiResponse` / `GlobalExceptionHandler` / `BackendMessageCatalog` / `BackendMessageResolver` の方針に従うこと
- 設定値をハードコードせず、application 設定または環境変数に外出しすること
- 状態は既存 enum / 状態コード / ジョブ状態管理に寄せること
- 複数更新がある場合はトランザクション境界を明示すること

## 4. 実装ポリシー

- 既存 API 互換性を極力維持する
- 変更は担当範囲に閉じる
- DB 変更が必要な場合は新規 migration を追加する
- テストを追加または更新する
- 実装後に、自分の担当範囲で何を変更したかを要約する

## 5. 禁止事項

- 他機能の大規模リファクタ
- 無関係な命名変更
- 既存の未対応課題を勝手にまとめて直すこと
- ログ・例外・レスポンスの直書き
- 他 Codex が触る可能性の高いファイルの不要な整形

## 6. 完了時に必ず報告する内容

- 変更ファイル一覧
- 実装した内容
- 追加した設定値
- 追加した DB 変更
- 追加 / 更新したテスト
- 未解決事項
