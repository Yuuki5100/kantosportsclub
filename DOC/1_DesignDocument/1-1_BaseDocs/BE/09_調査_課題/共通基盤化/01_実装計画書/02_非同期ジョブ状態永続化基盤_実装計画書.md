# 非同期ジョブ状態永続化基盤_実装計画書

## 1. 目的

帳票出力やダウンロード準備などの非同期ジョブ状態を、アプリ再起動後も参照できる永続化基盤へ移行する。

## 2. 対象範囲

- `appserver`
- `servercommon`
- ジョブ状態管理
- 生成ファイル保管

## 3. 現状

- ジョブ状態はメモリキャッシュで管理している
- 出力ストリームもメモリ保持している
- 再起動で状態が失われる

## 4. 到達目標

- ジョブ状態が DB で参照可能
- 出力ファイルがストレージで再取得可能
- ポーリング API が永続状態を返す
- 有効期限切れデータを掃除できる

## 5. 実装ステップ

### Step 1. 状態モデル整備

- ジョブ状態テーブル定義
- 生成物パス / MIME / 有効期限の保持

### Step 2. ジョブ実行側改修

- `ReportPollingRunner` などで状態を DB 更新
- ストレージにファイルを保存

### Step 3. 参照 API 改修

- ダウンロード準備状態 API の永続ストア参照化
- ダウンロード API のストレージ参照化

### Step 4. キャッシュ廃止または縮退

- `JobStatusCache*` の利用箇所を整理
- 段階移行するなら互換レイヤを挟む

### Step 5. クリーンアップ

- 期限切れ状態 / 生成物の削除ジョブ追加

## 6. 成果物

- DDL
- Repository / Service
- Runner 改修
- API 改修
- クリーンアップジョブ

## 7. 依存関係

- `StorageService`
- 帳票出力基盤
- 既存の `JobStatus` 運用

## 8. リスク

- 既存ポーリング仕様との互換性
- ストリーム即時返却からファイル参照返却への変化
- 大容量ファイルの保存先設計

## 9. テスト計画

- 正常な状態遷移
- 再起動後の再取得
- 期限切れ削除
- 失敗ジョブの再参照

## 10. 完了条件

- 再起動後もジョブ状態と生成物を取得できる
- メモリキャッシュに依存しない
