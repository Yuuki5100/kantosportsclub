# 通知再送制御基盤_実装計画書

## 1. 目的

`notify_queue` を使った通知配信を、再送上限とバックオフを備えた運用可能な通知基盤へ拡張する。

## 2. 対象範囲

- `appserver`
- `notify_queue`
- WebSocket 通知

## 3. 現状

- 未通知レコードを定期スキャンして送信している
- 失敗時は `retryCount++` のみ
- 上限回数と退避状態がない

## 4. 到達目標

- 再送上限を設定可能
- バックオフ制御がある
- 永久失敗状態を識別できる
- 運用監視に必要な情報を保持できる

## 5. 実装ステップ

### Step 1. 状態設計

- `PENDING`, `RETRY_WAIT`, `SENT`, `FAILED` などの状態定義
- 次回送信時刻、最終エラー内容の保持

### Step 2. スキャン条件改修

- 未通知かつ送信可能時刻到達分のみ取得
- 上限到達分の除外

### Step 3. 再送制御実装

- 線形または指数バックオフ
- 最大再送回数超過時の失敗確定

### Step 4. 運用補助

- 手動再送 API または管理処理
- 失敗件数の可視化

## 6. 成果物

- テーブル拡張
- `NotifyQueueScanService` 改修
- 必要に応じて管理 API

## 7. 依存関係

- `NotifyQueueRepository`
- `WebSocketNotificationService`

## 8. リスク

- eventType ごとに再送可否が異なる可能性
- クライアント切断時の扱いを整理する必要がある

## 9. テスト計画

- 正常通知
- 一時失敗からの再送成功
- 上限到達で失敗確定
- 手動再送

## 10. 完了条件

- 再送回数と待機時間を制御できる
- 失敗通知が運用上追跡できる
