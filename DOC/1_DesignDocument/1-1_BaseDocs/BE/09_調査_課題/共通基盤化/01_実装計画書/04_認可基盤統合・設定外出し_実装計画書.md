# 認可基盤統合・設定外出し_実装計画書

## 1. 目的

認可判定方式と権限定義の正本を整理し、案件追加時に再利用しやすい認可基盤へ統合する。

## 2. 対象範囲

- `appserver`
- `PermissionChecker`
- `CustomPermissionEvaluator`
- `PermissionConfigProvider`
- エンドポイント権限定義

## 3. 現状

- 判定方式が複数ある
- ドキュメント上も棲み分けが要確認
- 動的更新や再読込がない

## 4. 到達目標

- 権限制御の正本が明確
- 判定ルートが標準化される
- 権限定義を外部管理できる
- 設定変更を反映できる

## 5. 実装ステップ

### Step 1. 正本決定

- DB を正本とするか、設定ファイルを正本とするか決定
- `EndpointPermissionConfig` の役割を整理

### Step 2. 判定方式統合

- `@RequirePermission` と Method Security の関係を整理
- `PermissionChecker` / `CustomPermissionEvaluator` の責務を再定義

### Step 3. 権限モデル整備

- ロールレベル
- メニュー機能別要求レベル
- キャッシュ構造

### Step 4. 更新反映

- 起動時ロード
- 管理画面更新時の再読込またはキャッシュ破棄

## 6. 成果物

- 権限モデル設計
- Provider / Evaluator 改修
- テストケース整理

## 7. 依存関係

- `EndpointAuthorityMappingRepository`
- 認証基盤
- 管理画面または権限マスタ更新手段

## 8. リスク

- 既存 API の権限挙動差異
- 案件固有ロールとの整合

## 9. テスト計画

- ロール別アクセス
- カスタム権限
- 権限更新後の反映
- 未設定エンドポイント

## 10. 完了条件

- 権限設定の正本が一意に決まる
- 判定ルートがドキュメントと実装で一致する
