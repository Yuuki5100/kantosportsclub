# 認可基盤_状態コード定義書

## 1. 目的
認可基盤で使用するロールID・要求レベル・正本テーブルの意味を定義する。

## 2. ロールID（`UserRole`）

| 名称 | ID | 意味 |
| --- | --- | --- |
| `SYSTEM_ADMIN` | 1 | 管理者（`PermissionChecker` 上は常時許可） |
| `EDITOR` | 2 | 編集権限ロール |
| `VIEWER` | 3 | 参照権限ロール |
| `CUSTOM` | 4 | 個別権限（`role_permission`）で判定するロール |

## 3. 要求レベル（`statusLevelId` / `requiredLevel`）

| レベル | 意味（現行運用） |
| --- | --- |
| 1 | なし（操作不可） |
| 2 | 参照 |
| 3 | 変更（作成/更新/削除を含む） |

## 4. 正本テーブル

### 4-1. `role_permission`
- `roleId`, `permissionId`, `statusLevelId`
- ロールが保有する権限レベルの正本

### 4-2. `endpoint_authority_mapping`
- `method`, `url`, `menuFunctionId`, `requiredLevel`
- エンドポイント要求レベルの正本

## 5. 判定上の取り扱い
- 標準経路（`@RequirePermission`）は `role_permission` のみで判定
- Method Security補助経路は `endpoint_authority_mapping` と `role_permission` を組み合わせて判定
