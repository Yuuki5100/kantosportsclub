# フロントエンド モックモード 仕様書

## 1. 概要
フロントエンド内蔵モックモード（Demo Mode）の仕様を定義する。
環境変数1つで切替し、バックエンドやDBに接続せずにUI確認・デモを可能にする。
既存の認証・API共通層・通知/エラーハンドリング基盤を維持したまま動作する。

## 2. 目的
- バックエンド未接続でも画面確認を可能にする
- 画面/コンポーネントにモック分岐を持ち込まず、service層で切替する
- 既存基盤（認証、API、通知、エラー）を崩さずに実現する

## 3. 対象範囲
### 対象
- `src/api/services/v1` に存在する全API呼び出し
- `useApi` 経由で利用されるAPI
- 画面/コンポーネントから利用される service/hook

### 対象外
- WebSocketの完全モック
- バックエンド/DB変更
- 外部モックサーバー
- CRJ系API（要確認）

## 4. 切替仕様
### 環境変数
```
NEXT_PUBLIC_USE_MOCK_MODE=true
```
- `true` の場合のみモックモード有効
- 未設定または `false` の場合は通常モード
- 取得は `src/utils/envUtils.ts` の `isMockMode()` を使用

## 5. アーキテクチャ
### 5.1 レイヤ構成
```
[Page / Component]
        ↓
[hooks]
        ↓
[services/v1 (facade)]
  ├─ real/*  (実API)
  └─ mock/*  (モック)
        ↓
[apiService / apiClient]
```

### 5.2 分離方針
- `services/v1/*.ts` は facade として mock/real を切替
- モック未実装の場合は実APIへフォールバック
- 画面/コンポーネントでモック判定をしない

## 6. フォルダ構成
```
src/
  api/services/v1/
    real/
    mock/
    serviceSelector.ts
  mocks/
    auth/
    user/
    admin/
    role/
    notice/
    manual/
    systemSetting/
    report/
    errorCode/
    mailTemplate/
    import/
    files/
```

## 7. 切替ロジック
### 7.1 共通切替
- `serviceSelector.ts` に以下を定義する
  - `callWithMockFallback(mockFn, realFn)`
  - `selectHook(realHook, mockHook)`
  - `MockNotImplementedError`

### 7.2 フォールバック
- mock 側が `MockNotImplementedError` を投げた場合、real にフォールバックする
- 例: forgot-password / reset-password はモック未実装のため real へ切替

## 8. 認証モック
### 対象API
- login / logout / auth status / refresh

### セッション管理
- `sessionStorage` を使用
- `src/mocks/auth/handlers.ts` にてセッション保存/取得/削除を行う

### 返却形式
既存の `AuthStatusResponse` / `LoginData` に合わせる。
`rolePermissions` を返し、`ProtectedRoute` と `pageConfig` がそのまま動作すること。

## 9. APIモックの要件
- 返却型・構造は実APIと一致させる
- PromiseベースのI/Fを維持する
- 一覧系は必要に応じてページング/フィルタ/ソートを再現する
- 更新系は成功レスポンスを返す（過度な永続化は行わない）

## 11. 画面側ルール
- 画面/コンポーネントではモック判定をしない
- API呼び出しは `services/v1` 経由のみ
- `useApi` は実API用の基盤として維持（モックは service 層で切替）

## 12. 例外/制限
- CRJ系APIは対象外（要確認）
- forgot-password / reset-password はモック未実装のため実APIへフォールバック

## 13. 受入条件
- `NEXT_PUBLIC_USE_MOCK_MODE` で切替できる
- モックONでログイン・認証済み画面へ遷移できる
- モックOFFで実API動作に戻る
- 画面側にモック分岐がない
- 既存基盤（認証、API、通知、エラー）を迂回しない

## 14. 要確認事項
