# 管理者向けユーザー一覧画面 詳細設計書

## 1. 目的

本ドキュメントは、管理者向けユーザー一覧画面の詳細設計を定義する。
本画面は既存の認証基盤、API共通層、一覧表示コンポーネント、通知基盤を再利用して実装する。

---

## 2. 対象画面

- 画面名: 管理者向けユーザー一覧画面
- 想定URL: `/user/list` または `/admin/user/list`
- 対象利用者: 管理者権限を持つユーザー

※ URL は既存ルーティング設計に合わせて最終決定する。

---

## 3. 前提

- 認証済みユーザーのみアクセス可能
- 権限判定は `ProtectedRoute` と `pageConfig` を利用する
- API呼び出しは共通API層を経由する
- 一覧表示は `ControllableListView` を利用する
- エラー通知は共通エラーハンドラを利用する
- 成功/警告通知は Snackbar 基盤を利用する

---

## 11. イベント設計

## 11.1 初期表示

1. 画面へアクセス
2. `ProtectedRoute` により認証・認可確認
3. 検索条件初期値を生成
4. 一覧取得API呼び出し
5. 一覧を表示

## 11.2 検索ボタン押下

1. 入力値を検索条件に反映
2. page を 1 に戻す
3. 一覧取得API呼び出し
4. テーブル表示更新

## 11.3 クリアボタン押下

1. 検索条件を初期値へ戻す
2. page を 1 に戻す
3. 一覧取得API呼び出し
4. テーブル表示更新

## 11.4 ソート変更

1. sortKey / sortOrder を更新
2. 一覧取得API呼び出し
3. テーブル表示更新

## 11.5 ページ変更

1. page または size を更新
2. 一覧取得API呼び出し
3. テーブル表示更新

## 11.6 詳細遷移

1. 行クリック または 詳細ボタン押下
2. 対象 userId を取得
3. 詳細画面へ遷移

## 12. バリデーション設計

### 12.1 入力チェック

基本方針:
- 必須入力なし
- 桁数・形式制約がある場合のみ個別チェック

候補:
- メールアドレスが入力される場合は形式チェック
- 入力文字数上限は既存共通入力部品の制約に従う

※ 厳密な制約は要確認

---

## 13. エラーハンドリング設計

| 事象 | 対応 |
|---|---|
| 一覧取得失敗 | 共通エラーハンドラへ委譲 |
| 認証切れ | 共通認証制御へ委譲 |
| 権限不足 | リダイレクト + 通知 |
| マスタ取得失敗 | エラー通知表示 |
| 0件 | 正常系として空一覧表示 |

補足:
- APIエラーは `errorHandler` を利用する
- 画面個別で独自エラーハンドリングを持たない

---

## 14. 通知設計

| タイミング | 通知内容 | 種別 |
|---|---|---|
| 権限不足時 | アクセス権限がありません | warning |
| 一覧取得失敗時 | 一覧取得に失敗しました | error |
| マスタ取得失敗時 | 選択肢の取得に失敗しました | error |

---

## 15. 権限設計

- ページ単位で権限制御を行う

候補:
- ADMIN
- USER_MANAGEMENT

※ 正式な権限値は既存定義を参照して決定する

## 18. テスト観点

### 18.1 正常系
- 認証済み・権限ありで画面表示できる
- 初期表示で一覧取得できる
- 検索条件で絞り込みできる
- ソートできる
- ページングできる
- 詳細画面へ遷移できる

### 18.2 異常系
- 未認証時にログイン画面へ遷移する
- 権限不足時にリダイレクトする
- API失敗時に通知表示される
- 0件時に空一覧が表示される

### 18.3 境界値
- 1件のみ取得
- 最大件数取得
- 長い検索文字列入力
- メールアドレス形式不正

- API通信は共通API層を利用する
