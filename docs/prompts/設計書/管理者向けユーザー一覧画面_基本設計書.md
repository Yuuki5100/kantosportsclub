# 管理者向けユーザー一覧画面 基本設計書

## 1. 概要

本ドキュメントは、管理者がユーザーを検索・一覧表示し、詳細画面へ遷移できる機能の基本設計を定義する。

本機能は既存フロントエンド基盤（認証・API・一覧UI・通知）を再利用して実装することを前提とする。

---

## 2. 対象範囲

### 対象
- 管理者向けユーザー一覧画面
- 検索条件入力
- 一覧表示
- ソート・ページング
- 詳細画面遷移

### 対象外
- ユーザー編集
- ロール変更
- 新規作成
- CSV出力

---

## 3. 画面構成

```text
BasePage
├─ Header
├─ SideMenu
├─ Breadcrumb
├─ 検索条件エリア
└─ 一覧テーブルエリア
```

### 3.1 検索条件エリア

項目:
- ユーザー名
- メールアドレス
- ロール
- ステータス

操作:
- 検索ボタン
- クリアボタン


### 3.2 一覧テーブルエリア

表示項目:
- ユーザーID
- ユーザー名
- メールアドレス
- ロール
- ステータス

操作:
- 行クリックで詳細遷移
- カラムソート
- ページング

---

## 4. 画面遷移

```text
ユーザー一覧
  ↓（行クリック）
ユーザー詳細画面
```

---

## 5. 認証・認可

- ProtectedRouteを使用
- pageConfigで権限を定義

条件:
- 未認証 → ログイン画面へ遷移
- 権限不足 → 許可画面へリダイレクト + 通知

## 8. API設計

### 8.1 ユーザー一覧取得

```
GET /users
```

リクエスト:
- name
- email
- role
- status
- page
- size
- sort

レスポンス:
- list
- total

※エンドポイントは要確認

---

## 9. 処理フロー

### 初期表示
1. 認証確認
2. 一覧API呼び出し
3. テーブル表示

### 検索
1. 条件入力
2. API再取得
3. 一覧更新

### ソート/ページング
1. 状態更新
2. API再取得

---

## 10. エラーハンドリング

- APIエラーはerrorHandler経由
- Snackbarで通知

- 認証必須
