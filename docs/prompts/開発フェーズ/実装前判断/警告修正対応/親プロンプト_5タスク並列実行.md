## 2. 実行方針

- 1 タスク = 1 エージェント = 1 worktree = 1 作業ブランチ
- すべて `develop` 起点で開始する
- 各タスクは独立して進める
- 非自明な実装分岐がある場合のみ停止して判断を仰ぐ
- それ以外は止まらず最後まで実施する

## 5. エージェントへの共通指示

以下を各エージェントに前置きして渡すこと。

```text

必須ルール:
- 担当タスク以外に手を広げないこと
- 修正、動作確認、修正反映確認、コミット、ローカル develop へのマージ、作業ブランチと worktree の破棄まで完了すること

- 本文に必ず以下を入れる

  - `作業結果:`

作業結果には少なくとも以下を書くこと:
- 何を確認したか
- テストやログ確認結果
- 狙った warning がどう変化したか

- ローカル develop に `--no-ff` でマージ
- マージ後に branch 削除
- worktree 削除

最終報告に必ず含める項目:
- タスクID
- 作業ブランチ
- worktree
- 修正ファイル
- 実施した確認
- warning 解消確認結果
- コミットID
- develop へのマージ結果
- branch / worktree の削除結果
```
