## 5. 優先順位の判断基準

優先度「高」とする条件:

- Next.js / React / Spring Boot の将来バージョンでエラー化しやすい
- 実行ログの可読性を大きく下げている
- 修正コストが低く、効果が大きい

優先度「中」とする条件:

- 直ちに失敗しないが、保守負債として増殖しやすい
- ファイル数は多いが、同一パターンで横展開できる

優先度「低」とする条件:

- 動作影響は小さい
- ライブラリやテスト基盤の移行方針を決めてから着手した方が安全

## 10. 実行時の注意点

- `_document` の `<title>` 修正は SEO や画面タイトルの表示位置に影響するため、共通レイアウトまたは各ページでの title 設定方針を先に統一する
- `react-hooks/exhaustive-deps` は warning を消すこと自体を目的にせず、再フェッチや再描画の挙動確認を必ずセットで行う
- `@MockBean` 置換は BE 全域に波及するため、モジュール単位で小さく分割する
- Mockito agent 対応は JDK / Surefire / Mockito の組み合わせ確認が必要なため、先に検証用ブランチで試す
